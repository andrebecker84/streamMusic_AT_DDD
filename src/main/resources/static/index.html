Perguntas frequentes.

Tudo que você precisa saber sobre o projeto StreamMusic e sua arquitetura.

O que é o StreamMusic?

Plataforma acadêmica de streaming de música desenvolvida no curso DR2 — Design Patterns & DDD do Instituto Infnet. Demonstra na prática padrões de projeto como Strategy e Repository, arquitetura DDD com bounded contexts, princípios SOLID e Clean Code — tudo em Spring Boot 4 + Java 25.

Quais tecnologias são utilizadas?

Backend: Spring Boot 4.1 · Java 25 · H2 (banco em memória) · Redis (cache) · Springdoc OpenAPI · Lombok. Front-end: HTML, CSS e JavaScript puros — sem framework — servidos pelo próprio Spring Boot.

O que é DDD e como é aplicado aqui?

Domain-Driven Design coloca o modelo de negócio no centro do software. O projeto possui 4 bounded contexts independentes: Conta, Assinatura, Transação + Antifraude e Música. Cada contexto tem seus próprios Aggregates, Value Objects, repositórios e serviços — integrados por referência de ID, nunca por objetos diretos.

Como funciona o sistema antifraude?

Usa o Strategy Pattern: cada regra é uma classe independente que implementa RegraAntifraude. O ServicoAntifraude (Domain Service) injeta a lista e executa todas. Novas regras são adicionadas criando uma nova classe @Component — sem alterar nada existente (Princípio Open/Closed).

Quais regras antifraude existem?

São 5 regras: cartão-de-crédito-inválido (sem cartão cadastrado), cartão-não-ativo (cartão bloqueado), plano-ativo-inválido (sem assinatura ativa), alta-frequência-pequeno-intervalo (máx. 3 transações em 2 min) e transação-duplicada (mesmo valor + comerciante em 2 min).

Posso ter mais de um plano ativo?

Não. A regra de negócio limite-de-plano-ativo garante exatamente um plano ativo por conta. Ao tentar assinar um segundo plano, a API retorna erro 422 com a violação informada. Cancele o plano atual antes de assinar outro.

O que são Aggregates e Value Objects?

Aggregates são entidades com identidade UUID única — Account, Transaction, Playlist. Value Objects são imutáveis e sem ID próprio — CreditCard, Money, Merchant. A distinção define os limites de consistência do domínio e é central no DDD de Eric Evans.

Como explorar a API?

Acesse o Swagger UI em /swagger-ui.html para testar todos os endpoints interativamente. O H2 Console em /h2-console permite inspecionar o banco em memória — use a JDBC URL jdbc:h2:mem:streaming sem senha.

O projeto tem testes automatizados?

Sim. Testes de integração com MockMvc seguindo a metodologia AAA (Arrange / Act / Assert) cobrem os principais fluxos: criação de conta, assinatura de plano, autorização de transação e todas as regras antifraude. Execute com mvn test.

Posso rodar com Docker?

Sim. Execute docker compose up --build para subir a API com Redis real + Redis Commander na porta 8081. Sem Docker, a aplicação usa cache em memória automaticamente — basta mvn spring-boot:run.
